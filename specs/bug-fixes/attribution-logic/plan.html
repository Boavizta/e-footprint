Attribution-logic revamp — Implementation plan

Status: Plan — under review · 2026-06-09 · Spec: analysis.md + edge-analysis.md + settled design decisions (below). Scope: end-game structure only; migration steps deferred. No backward compatibility.

1. Approach

Replace the generic per-object repartition machinery in ModelingObject with an explicit, source-anchored attribution layer that lives in the domain. The whole layer is a pure, lazy, read-time projection over the settled calculated-attribute graph. Three roles, one engine:

Sources own physics (one cell-evaluator each) · Containers own dimensionless structural weights · One generic engine climbs the containment chain, folds to any level, and handles skip / exclude / cross-column flows. Every flow is natively {source: value}.

The cell-evaluator: why it stays flat, never a nested dict

Each source exposes a single recipe for "my footprint at cell (container, up)". Stored state stays flat single-key {container: hourly} — the shape of today's *_per_job dicts:

  • Neutral-CI sources (Server energy = server-country CI, constant; Storage fabrication): store the flat leaf-split Server→Job = {job: hourly}; the engine redistributes to any finer/coarser container by dimensionless container weights — no pre-join.
  • Country-dependent sources (Network, Device): store no joint dict — expose a physics function (energy_footprint_for_data_volume_and_usage_pattern(data, up)); the per-cell data volume is container-owned. The join exists only transiently inside the fold, with CI[up] kept inside.

The only inherently 2-keyed object — a cross-column link value (e.g. Step↔UP) — is computed on demand for the visible adjacent pair and memoized per render, never stored on the model. So no deep nesting ever lands in a source class, and explainability labels stay clean.

Each is one method — "one atom function to review per source". Everything above the leaf is the shared engine.

1.2 The regroup engine: one chain, one per-edge weight

Every Source→ancestor and container↔container flow is the same operation: climb the containment chain, carrying one weight per edge. Web carries occurrence counts wherever jobs or steps repeat — at Job↔Step (a job runs N times in a step) and Step↔UJ (a step recurs within, or is shared across, journeys); only the UP and Country regroups are truly weight-1 (each UP has one journey, one country). Edge carries the occurrence-ratio counts throughout. Same engine, both sides.

Reusable physics living in the engine / sources (single source of truth)
  • energy_footprint_for_data_volume_and_usage_pattern(data, up) on Network — data→carbon, called by every network cell.
  • cumsum-with-dumps — factored out of Storage (now used for per-job, per-step, per-up, per-RSN cumulative needs).
  • binding-resource raw[h] + idle/dynamic energy split — Server (also feeds the eager energy total).
  • occurrence-ratio / topology counts — container-owned, consumed as the per-edge weight.

1.3 Caching & invalidation

  • Entire attribution layer = lazy @cached_property → pay-on-render; zero cost at build and on un-rendered updates.
  • Auto-flush every cached property (discover cached_property descriptors via the class MRO, memoized per class). Drops the manual _attributed_footprint_cached_property_names registry and its per-class overrides.
  • Flush is a flat system-wide sweep over all_linked_objects, once at the end of every ModelingUpdate and after the initial build. No traversal-graph dependency.
  • Invariant: calculated attributes never read attribution cached properties. This one-way rule is what makes wholesale lazy flush correct.

1.4 Skip & exclude = folds, not rescaling

Because flows are absolute, sibling-independent, and additively decomposed, both operations are read-side folds over the cached cells — no normalization anywhere:

  • Skip a column = read the source's edge to the next visible level directly (the engine already computes every level). No spacer nodes.
  • Exclude a source (the common case, e.g. Server) = drop its term from each flow's per-source sum. No rescale.
  • Exclude an intermediate / subtree = additively re-aggregate coarser levels from surviving finer cells.

This deletes the entire resolve_attributed_footprint_per_source / _merge_rescaled apparatus. As a payoff, footprint is traceable to impact sources at every container level (not just leaves), and per-source-colored flows become trivial if added later.

1.5 Worked example — ServerBase

Server is neutral-CI (its own country's CI, constant across patterns), so its leaf is Job and its entire attribution surface is a flat {job: hourly} leaf-split; every coarser/edge level is engine-generated. The two streams (provisioned ∝ nb_of_instances, dynamic ∝ raw_nb_of_instances) and the idle/load energy split already exist — only the per-job binding-resource weights are new.

Full shape: server_base_sketch.py (illustrative, analysis.md-style bodies). At a glance:

  • deleted — job_repartition_weights, the two *_impact_repartition_weights properties, the inherited generic *_impact_repartition*.
  • kept (eager) — nb_of_instances, raw_nb_of_instances, instances_energy (idle+load), the two footprint totals — unchanged.
  • new physics (cached) — binding_raw[h], dynamic_share_per_job, provisioned_share_per_job (per-tier, on-prem vs autoscaling).
  • new atom (cached) — two flat {job: hourly} leaf-splits + attribution_leaf = JobBase / attribution_neutral_ci = True. That is the entire Tier-B surface.

So the reviewable Server-specific code is exactly: two stream weights + two leaf-split dicts. The ~7 per-level / edge flows analysis.md lists for Server are not methods here — they are the same engine fold applied with a different Tier-A weight dict. Storage mirrors this (neutral, leaf = Job, two streams); Network/Device differ only in being country-dependent — a footprint_at(leaf, up) physics fn with CI[up] inside, instead of a flat leaf-split. The country-dependent counterpart is sketched in device_sketch.py.

1.6 The engine — core/attribution/

A single module of pure functions over the source/container contract — the only cross-cutting attribution code, and not a ModelingObject. Sources and containers expose a tiny contract; the engine owns the climb, the folds, and skip/exclude. One primitive (per_level) does the work; the public folds are thin reads over it.

Full shape: engine_sketch.py. The essentials:

  • contract — source declares attribution_leaf + attribution_neutral_ci; neutral sources expose a flat {leaf: hourly} split, country-dependent ones a footprint_at(leaf, up) physics fn; containers expose weight_for(level) + attribution_total; topology exposes level_ancestor / nodes_at.
  • primitive — per_level(source, level, phase): neutral = weighted_redistribution (leaf-split × dimensionless occurrence weight); country-dep = group_cells_by_ancestor (sum footprint_at cells, CI inside, by level_ancestor). Both fleshed out in the sketch. Cached, flushed wholesale.
  • folds — node_total(node, level, phase, exclude) = Σ surviving sources; flow(finer, coarser, phase, exclude) = the cross-column link value: for a clean pair (UP→Country, UJ→UP) it's just the pattern's node_total; for a many-to-many pair (Job↔Step, or Step↔UP/Job↔UP when a column is skipped) it resolves the joint cell per source via a container Tier-A joint weight, on demand. Skip = pick the level; exclude source = drop the term; exclude subtree = additive re-sum. No normalization anywhere.

The renderer becomes a column-walk (sketched in sankey_sketch.py): for each adjacent pair of visible columns it calls flow(), and node_total() for node sizes — passing the excluded-source set straight through. Skipping a column is just leaving its level out of the visible set; the adjacent columns link directly. No recursion, no spacer nodes, no resolve_*. Adding a new source later means implementing the contract on it and nothing else; adding a new level means one entry in the topology + the matching Tier-A weight.

3. Cross-cutting concerns

  • Tests: conservation invariants are the backbone — every "Conserves / Σ = total" claim in the analysis docs becomes a property test (Σ over tiers = per-job; Σstep = full cumulative; Σuj = Σup; Σcountry = source total; Σ over a pattern's needs = device total). Plus per-source-exclusion and column-skip equivalence tests. Existing Sankey/integration tests are re-pointed at the new flows.
  • Serialization: attribution is lazy cached, not a calculated attribute → not serialized. Removing the repartition-weight calculated attributes is a round-trip simplification (recomputed on load), no schema/version change expected.
  • Docs: architecture.md (attribution boundary, resolved-attribution, cached-property registry sections) and conventions.md (repartition-weight preferences) need updating to the new model.

4. Risks

  • Server per-tier provisioned stream — the single subtlest atom (on-prem vs autoscaling, raw[h] > k−1). Isolate as a tested helper.
  • Cell-evaluator ⇄ engine boundary — risk of leaking source physics into the engine or container weights into sources. Keep the contract minimal: source returns a cell value; engine never knows a source's formula.
  • Auto-flush global invariant — commits the codebase to "every @cached_property is a flushable read-time projection". True today; must be stated as a rule so it stays true.
  • Renderer rewrite — column-walk + on-demand cross-column flows must reproduce current totals before old paths are deleted (validated by conservation tests, not by keeping a façade).

5. Alternatives considered

  • Keep generic repartition in ModelingObject — rejected: it's the framework-layer leak and the root cause of column-skip pain (original sin #2).
  • Materialize finest joint atoms (source, step, up) as stored dicts — rejected: deep nesting, wrecks explainability; factoring into flat leaf-split × container weights is leaner.
  • Bundle each flow's sources into one Step→UP function — rejected: would import every source's physics into the container and block source exclusion / source-traceability / future coloring.
  • Eager calculated attributes for the fine breakdowns — rejected: only consumed at render; lazy cached keeps them pay-on-read.
  • Per-edge incremental cache invalidation — rejected: fragile; wholesale flush is correct and simple (caching overhaul is coming separately).

6. Constitutional notes

  • §1.1 three-layer separation — net positive: attribution moves out of abstract_modeling_classes/ into core/, relieving the documented back-edge. New engine imports only upward.
  • §1.2 no backward compatibility — explicitly honored: old paths are deleted, not shimmed.
  • §1.3 leanness — net deletion of generic machinery; per-source code is one evaluator each.
  • §1.4 doc-as-code — each cell-evaluator / Tier-A weight needs an update_*-style docstring as SSOT.
  • §2.3 round-trip — preserved (see §3). No §4 rules touched → no amendment required.
